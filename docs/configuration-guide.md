# Configuration reference

> Generated by `fno config schema --markdown` from the Pydantic `SettingsModel` (the single source of truth). Do not edit by hand; edit the model + `cli/src/fno/config/registry.py` and regenerate.

Keys live in a flat `config.toml` (`.fno/config.toml` project-local, `~/.fno/config.toml` global). A dotted key like `branch.prefix` is `prefix` under `[branch]`. For a copy-paste template with every key at its default, see [config.example.toml](config.example.toml); read or set individual keys with `fno config get <key>` / `fno config set <key> <value>`.
