# Speculate Variation Prompts

Creative direction catalog for speculative parallel implementations.

## UI Feature Variations

| Variation | Constraint |
|-----------|-----------|
| v1 | Data-dense: maximize information per viewport, tables, compact spacing |
| v2 | Visual: card-based layout, generous whitespace, imagery and icons |
| v3 | Minimal: progressive disclosure, essential info only, clean lines |
| v4 | Playful: animations, micro-interactions, personality in the UI |
| v5 | Enterprise: formal, information hierarchy, keyboard-navigable |

## Architecture Variations

| Variation | Constraint |
|-----------|-----------|
| v1 | Server-first: server components, server actions, minimal client JS |
| v2 | Client-rich: client components, optimistic updates, real-time sync |
| v3 | Hybrid: server render with progressive enhancement |

## API Design Variations

| Variation | Constraint |
|-----------|-----------|
| v1 | REST: resource-oriented, conventional HTTP, HATEOAS links |
| v2 | tRPC: type-safe, colocated, end-to-end inference |
| v3 | GraphQL: flexible queries, subscription support, schema-first |

## State Management Variations

| Variation | Constraint |
|-----------|-----------|
| v1 | URL state: search params, shareable, server-renderable |
| v2 | Client store: Zustand/Jotai, reactive, optimistic |
| v3 | Server cache: TanStack Query, stale-while-revalidate |

## Data Storage Variations

| Variation | Constraint |
|-----------|-----------|
| v1 | Relational: PostgreSQL, normalized tables, foreign keys |
| v2 | Document: MongoDB/Firestore, denormalized, flexible schema |
| v3 | Edge-first: SQLite/Turso, read replicas near users |
